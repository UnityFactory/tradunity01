Que sont les AssetBundles?

Les AssetBundles sont une collection d'assets, emballés pour le chargement lors de l'exécution. Avec les AssetBundles, vous pouvez charger et décharger dynamiquement un nouveau contenu dans votre application. Les AssetBundles peuvent être utilisés pour des DLC.

A quoi servent t'ils?

Ils peuvent être utilisés pour réduire la quantité d'espace sur le disque utilisé par votre jeu, lors de la première installation. Il peut également être utilisé pour ajouter du nouveau contenu à un jeu déjà publié.

Comment créer un AssetBundle?

Pour créer un AssetBundle vous devez utiliser la class editor BuildPipeline. Tous les scripts utilisant les classes editor doivent être placés dans un  dossier nommé Editor, n'importe où dans le dossier Assets. Voici un exemple d'un tel script en C #:

using UnityEngine;
using UnityEditor;

public class ExportAssetBundles {
    [MenuItem("Assets/Build AssetBundle")]
    static void ExportResource () {
        string path = "Assets/myAssetBundle.unity3d";
        Object[] selection = Selection.GetFiltered(typeof(Object), SelectionMode.DeepAssets);
        BuildPipeline.BuildAssetBundle(Selection.activeObject, selection, path, 
                                       BuildAssetBundleOptions.CollectDependencies 
                                     | BuildAssetBundleOptions.CompleteAssets);
    }
}

Un script editor n'a pas besoin d'être appliqué à un gameobject, il est plutôt utilisé par l'éditeur. Cet exemple précédent créera un nouvel élément dans le menu «Assets» de votre éditeur appelé "Build AssetBundle".

Pour utiliser cet exemple:

  • Créer un script C # nommé ExportAssetBundles.cs, à l'intérieur d'un dossier nommé Editor, dans la vue du projet.
  • Sélectionnez le ou les Assets dans votre dossier de projet que vous souhaitez mettre  dans l' AssetBundle.
  • Sélectionnez Construire AssetBundle dans le menu Assets. Cliquez sur "Save" pour créer l'AssetBundle.
  • La première ligne de la fonction  ExportResource définit le chemin de votre AssetBundle.
  • La ligne suivante définit que les objets sélectionnés dans la fenêtre Projet pourront être transformé en un AssetBundle.

La fonction BuildAssetBundle est la ligne qui crée le AssetBundle et l'enregistre à l'emplacement spécifié. Le premier paramètre spécifie le mainAsset, qui est un asset spécial qui peut être obtenu directement avec la propriété mainAsset lors du chargement d'un asset de l'AssetBundle. Il n' est pas obligatoire de mettre un Asset principal, si cela ne va pas être utilisé, vous pouvez utiliser null pour le paramètre. Le deuxième paramètre est le tableau d'objets qui feront le AssetBundle. Le troisième paramètre est l'emplacement sur le disque ou l'AssetBundle sera sauvegardé. Le dernier paramètre sont les drapeaux de construction ou des options utilisées lors de la construction de l'AssetBundles. OU binaire ('|') combine les options pour construire les AssetBundles. Dans cet exemple BuildAssetBundleOptions.CollectDependencies inclu d'autres actifs référencés par les objets et BuildAssetBundleOptions.CompleteAssets assurez-vous que les actifs sont complètement inclut La ligne suivante définit  les objets sélectionnés dans la fenêtre Projet pour être transformé en un AssetBundle.

La construction d'un AssetBundles devraient être une étape de pré-publication qui arrive une seule fois et en un seul appel de fonction, par exemple, avec un élément de menu qui construit tous les AssetBundles. Comme vous développez votre application, vous devriez écrire des scripts auxiliaires qui peuvent construire tous vos AssetBundles pour une plate-forme cible d'un simple clic ou batchmode sans intervention de l'utilisateur.

 

Il ya aussi une autre fonction pour créer AssetBundles. Vous pouvez en lire plus à ce sujet ici.

Comment puis-je utiliser un AssetBundle?

Il ya deux principales étapes lorsque vous travaillez avec AssetBundles. La première étape consiste à télécharger l'AssetBundle depuis un serveur ou un disque. Cela se fait avec la class WWW. La deuxième étape consiste à charger les assets de l'AssetBundle, pour être utilisé dans l'application. Voici un exemple de script C #:

using UnityEngine;
using System.Collections;

public class BundleLoader : MonoBehaviour{
    public string url;
    public int version;
    public IEnumerator LoadBundle(){
        using(WWW www = WWW.LoadFromCacheOrDownload(url, version){
            yield return www;
            AssetBundle assetBundle = www.assetBundle;
            GameObject gameObject = assetBundle.mainAsset as GameObject;
            Instantiate(gameObject );
            assetBundle.Unload(false);
        }
    }
    void Start(){
        StartCoroutine(LoadBundle());
    }
}

Ce script est ajouté à un gameobject comme composant. Le AssetBundle est chargé de la manière suivante:

  • Les valeurs de l'URL et de la variable version doivent être mis dans l'inspecteur avant l'exécution de ce script. L'URL est l'emplacement du fichier de AssetBundle, généralement un serveur sur Internet. Le numéro de version permet au développeur d'associer un numéro à l'AssetBundle lorsqu'il est écrit dans le cache dans le disque. Lorsque vous téléchargez un AssetBundle Unity va vérifier si le fichier existe déjà dans le cache. Si c'est le cas il compare la version stockée de asset à la version requise. Si elle est différente alors l'AssetBundle sera retélécharger. Si ce est la même, alors il va charger l'AssetBundle a partir du disque et éviter d'avoir à retélécharger le fichier. Se référer à la fonction de WWW.LoadFromCacheOrDownload dans la référence de script pour plus d'informations sur ces paramètres.
  • Lorsque la fonction Start de ce script est appelé, il va commencer le chargement de l'AssetBundle en appelant la fonction de Coroutine.The function will yield on the WWW object as it downloads the AssetBundle. By using this, the function will simply stop in that point until the WWW object is done downloading, but it will not block the execution of the rest of the code, it yields until it is done. Only up to one AssetBundle download can finish per frame when they are downloaded with WWW.LoadFromCacheOrDownload
  • Une fois l'objet de WWW a téléchargé le fichier AssetBundle, la propriété .assetBundle est utilisé pour récupérer un objet AssetBundle. Cet objet est l'interface pour charger des objets à partir du fichier de AssetBundle.
  • Dans cet exemple, une référence à un préfabriquée dans l'AssetBundle est extrait de l'AssetBundle utilisant la propriété .mainAsset. Cette propriété est définie lors de la construction de l'AssetBundle passant un objet comme premier paramètre. Le principal atout dans l'AssetBundle peut être utilisée pour stocker une TextAsset avec une liste d'objets à l'intérieur de l'AssetBundle et toute autre information à leur sujet.

S'il vous plaît noter que pour simplifier l'exemple précédent ne fait pas tous les contrôles de sécurité. Regarder le code ici pour un exemple plus complet.

Comment puis-je utiliser les AssetBundles dans l'éditeur?

Comme la création d'applications est un processus itératif, vous aurez très probablement à modifier vos assets de nombreuses fois, ce qui nécessiterait la reconstruction des AssetBundles après chaque changement pour pouvoir les tester. Même si il est possible de charger les AssetBundles dans l'éditeur, ce n'est pas une methode de travail recommandée. Au lieu de cela, tout en testant dans l'éditeur, vous devez utiliser la fonction d'aide Resources.LoadAssetAtPath pour éviter d'avoir à utiliser et reconstruire les AssetBundles. La fonction vous permet de charger l'Asset comme si il était en cours de chargement à partir d'un AssetBundle, mais ignore le processus de construction et vos Assets sont toujours à jour.

Voici un script d'exemple, que vous pouvez utiliser pour charger vos Assets en fonction de si vous exécutez dans l'éditeur ou non. Mettez ce code dans un script C# nommé AssetBundleLoader.cs:

using UnityEngine;
using System.Collections;

public class AssetBundleLoader {
    public Object Obj; // The object retrieved from the AssetBundle
   
    public IEnumerator LoadBundle<T> (string url, int version, string assetName, string assetPath) where T : Object {
        Obj = null;

#if UNITY_EDITOR
        Obj = Resources.LoadAssetAtPath(assetPath, typeof(T));
        if (Obj == null)
            Debug.LogError ("Asset not found at path: " + assetPath);
        yield break;

#else

        WWW download;
        if ( Caching.enabled ) { 
            while (!Caching.ready)
                yield return null;
           download = WWW.LoadFromCacheOrDownload( url, version );
        }
        else {
            download = new WWW (url);
        }

        yield return download;
        if ( download.error != null ) {
            Debug.LogError( download.error );
            download.Dispose();
            yield break;
        }

        AssetBundle assetBundle = download.assetBundle;
        download.Dispose();
        download = null;

        if (assetName == "" || assetName == null)
        Obj = assetBundle.mainAsset;
        else
            Obj = assetBundle.Load(assetName, typeof(T));
       
        assetBundle.Unload(false);

#endif
    }
}

Nous pouvons maintenant utiliser le script AssetBundleLoader pour charger un Asset d'un AssetBundle si nous lançons l'application construite ou charger asset directement à partir du dossier de projet en cas d'exécution dans l'éditeur:

using UnityEngine;
using System.Collections;

public class ExampleLoadingBundle : MonoBehaviour {
    public string url = "http://www.mygame.com/myBundle.unity3d"; // URL where the AssetBundle is
    public int version = 1; // The version of the AssetBundle

    public string assetName = "MyPrefab"; // Name of the Asset to be loaded from the AssetBundle
    public string assetPath; // Path to the Asset in the Project folder

    private Object ObjInstance; // Instance of the object
    void Start(){
        StartCoroutine(Download());
    }

    IEnumerator Download () {
        AssetBundleLoader assetBundleLoader = new AssetBundleLoader ();
        yield return StartCoroutine(assetBundleLoader.LoadBundle <GameObject> (url, version, assetName, assetPath));
        if (assetBundleLoader.Obj != null)
            ObjInstance = Instantiate (assetBundleLoader.Obj);
    }

    void OnGUI(){
        GUILayout.Label (ObjInstance ? ObjInstance.name + " instantiated" : "");
    }
}

Le script doit être enregistré dans un fichier nommé ExampleLoadingBundle.cs dans le dossier Assets. Après avoir défini les variables publiques à leurs valeurs correctes et en l'exécutant, il utilisera la class AssetBundleLoader pour charger un Asset. Il est ensuite instancié et sera instantié en utilisant l'interface graphique.

Comment puis-je mettre en cache les AssetBundles?

Vous pouvez utiliser  WWW.LoadFromCacheOrDownload  qui prend automatiquement soin de sauvegarder vos AssetBundles sur le disque. Soyez conscient que sur le Webplayer vous êtes limité à 50 Mo au total (partagé entre tous webplayers). Vous pouvez acheter une licence de mise en cache distincte pour votre jeu si vous avez besoin de plus d'espace.

Si vos AssetBundles sont stockés dans le dossier StreamingAssets en tant qu'AssetBundles incompressé, vous pouvez utiliser AssetBundle.CreateFromFile pour référencer l'AssetBundle sur le disque Si les AssetBundles dans le dossier StreamingAssets sont compressés, vous aurez besoin d'utiliser WWW.LoadFromCacheOrDownload pour créer une copie non compressée du AssetBundle dans le cache.

AssetBundles sont multi-plateforme?

Les AssetBundles sont compatibles entre certaines plates-formes. Utilisez le tableau suivant comme guide.
La compatibilité des plate-forme pour AssetBundles
Standalone Webplayer iOS Android
Editor Y Y Y Y
Standalone Y Y
Webplayer Y Y
iOS Y
Android Y

For example, a bundle created while the Webplayer build target was active would be compatible with the editor and with standalone builds. However, it would not be compatible with apps built for the iOS or Android platforms.

How are assets in AssetBundles identified?

When you build AssetBundles the assets are identified internally by their filename without the extension. For example a Texture located in your Project folder at “Assets/Textures/myTexture.jpg” is identified and loaded using “myTexture” if you use the default method. You can have more control over this by supplying your own array of ids (strings) for each object when Building your AssetBundle with BuildPipeline.BuildAssetBundleExplicitAssetNames.

Can I reuse my AssetBundles in another game?

AssetBundles allow you to share content between different games. The requirement is that any Assets which are referenced by GameObjects in your AssetBundle must be included in the AssetBundle. To make sure the referenced Assets are included in the AssetBundle when they are built you can pass the BuildAssetBundleOptions.CollectDependencies option.

Will an AssetBundle built now be usable with future versions of Unity?

AssetBundles can contain a structure called a type tree which allows information about asset types to be understood correctly between different versions of Unity. On desktop platforms, the type tree is included by default but can be disabled by passing the BuildAssetBundleOptions.DisableWriteTypeTree to the BuildAssetBundle function. Webplayers intrinsically rely on the type tree and so it is always included (ie, the DisableWriteTypeTree option has no effect). Type trees are never included for mobile and console asset bundles and so you will need to rebuild these bundles whenever the serialization format changes. This can happen in new versions of Unity. (Except for bugfix releases) It also happens if you add or remove serialized fields in monobehaviour’s that are included in the asset bundle. When loading an AssetBundle Unity will give you an error message if the AssetBundle must be rebuilt.

How can I list the objects in an AssetBundle?

You can use AssetBundle.LoadAll to retrieve an array containing all objects from the AssetBundle. It is not possible to get a list of the identifiers directly. A common workaround is to keep a separate TextAsset to hold the names of the assets in the AssetBundle.
